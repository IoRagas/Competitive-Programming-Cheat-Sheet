QUICK DECISION TREE
====================

Problem seems hard?
├─ Can it be solved with greedy? → Try greedy
├─ Need to find optimal subset? → Try DP or bitmask
├─ Need to optimize over range? → Try segment tree
├─ Need to match pattern? → Try KMP or hashing
├─ Is it graph problem? → BFS/DFS/Dijkstra
└─ Else: rethink the problem

TLE - Execution too slow:
├─ Check loop complexity
├─ Use faster I/O (fast I/O flag)
├─ Optimize inner loop
├─ Better algorithm (avoid brute force)
└─ Reduce constant factors

WA - Wrong answer:
├─ Check sample manually
├─ Edge cases (empty, single, max values)
├─ Off-by-one errors
├─ Integer overflow
└─ Output format

ALGORITHM COMPLEXITY TABLE
===========================
Algorithm              Complexity      Best For
Linear Search          O(n)            Unsorted data
Binary Search          O(log n)        Sorted array
Bubble Sort            O(n²)           Small arrays
Quick Sort             O(n log n)      Average case
Merge Sort             O(n log n)      Worst case guarantee
Heap Sort              O(n log n)      In-place required
BFS                    O(V+E)          Shortest path (unweighted)
DFS                    O(V+E)          Connectivity
Dijkstra               O((V+E)log V)   Positive weights
Floyd-Warshall        O(V³)           All pairs (small V)
Kruskal MST           O(E log E)       Sparse graphs
Union-Find (Kruskal)   O(E α(V))       Efficient unions
0/1 Knapsack          O(nW)           Optimal selection
Matrix Chain Mult     O(n³)           Order matters
